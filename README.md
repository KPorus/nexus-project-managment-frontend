# Nexus Project Managment

A robust, role-based administration and project management dashboard built with React 19, Redux Toolkit, and Tailwind CSS. This system is designed to handle secure user onboarding via email invitations, project lifecycle management, and administrative oversight with granular permission controls.

## ğŸ¯ Project Purpose & Objectives

The primary goal of Nexus is to provide a secure and scalable foundation for internal tools where user access needs to be strictly controlled.

**Key Objectives:**

- **Secure Onboarding:** Eliminate open registration. Users can only join via signed email invitations generated by administrators.
- **Role-Based Access Control (RBAC):** Ensure sensitive data and actions (like User Management) are restricted to specific roles (Admin) while allowing operational flexibility for Managers and Staff.
- **Data Integrity:** Implement "Soft Delete" patterns to preserve data history while maintaining a clean user interface.
- **Session Security:** Handle JWT authentication with automatic silent token refreshing to provide a seamless user experience without compromising security.

## ğŸ’¬ Architectural Discussion

### Service Layer Pattern

The application uses a dedicated `apiService` layer wrapping a core `request` utility. This decouples the UI components from specific API implementation details. The `request` utility handles the complexity of headers and the specific logic for intercepting expired tokens to trigger a refresh flow.

### Redux Thunks

We utilize Redux Async Thunks for side effects. This keeps our components clean and focuses on UI logic, while the state management layer handles loading states, error catching, and data normalization.

### Soft Delete Strategy

Instead of physical `DELETE` requests that permanently wipe data, the delete action typically triggers a status update ( `isDeleted: true`). This is crucial for enterprise applications where data recovery and audit trails are mandatory requirements.
